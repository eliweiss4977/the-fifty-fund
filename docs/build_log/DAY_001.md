## Claude Code Session 1

**Date:** April 15, 2026
**Session goal:** Create all core agent files — trading logic, X posting, Substack engine, and unified runner.

### Files Created

#### `agent/algomind_agent.py`
The heart of the fund. This file:
- Connects to **Alpaca** (paper or live) via `alpaca-trade-api` for real trade execution using fractional/notional orders.
- Uses **Claude AI** (`claude-sonnet-4-20250514`) as the sole decision-maker — every BUY/SELL/HOLD is Claude's call, not a hardcoded rule.
- Fetches 30 days of OHLCV data for all 9 tickers via **yfinance** and computes RSI-14, 1-day price-change %, and volume.
- Sends every trade + daily summaries to **Telegram** (silently skips if not configured).
- Optional **email** notifications via Gmail SMTP — silently skipped if `EMAIL_*` env vars are absent.
- Standalone scheduler: trade cycle every 30 min during NYSE hours (9:30am–4:00pm ET, Mon–Fri), daily summary at 4:05pm ET.
- Key exports for integration: `run_trade_cycle()`, `send_daily_summary()`, `get_portfolio()`, `fetch_market_data()`, `send_telegram()`.
- **Status:** Complete. Requires `ALPACA_API_KEY`, `ALPACA_SECRET_KEY`, `ANTHROPIC_API_KEY`, and optionally `TELEGRAM_*` / `EMAIL_*` env vars.

#### `agent/x_poster.py`
The social media voice of AlgoMind. This file:
- Posts **every trade decision** with full AI reasoning to X via **tweepy v4** (Twitter API v2).
- Posts **morning market outlook** once per trading day at open — highlights top mover, oversold/overbought tickers.
- Posts **EOD summary** tweet with portfolio value, P&L, and position count.
- Posts **weekly recap vs S&P 500** every Friday — fetches SPY live for comparison.
- Auto-posts **milestone tweets** when the fund hits: first trade, first profit, +10%, +25%, +50%, +100%.
- Milestone state persisted to `milestones_hit.json` at repo root (JSON file, survives restarts).
- Gracefully disabled if X API credentials are absent (logs a warning, returns False from all post functions).
- **Status:** Complete. Requires `X_API_KEY`, `X_API_SECRET`, `X_ACCESS_TOKEN`, `X_ACCESS_SECRET` env vars.

#### `agent/substack_engine.py`
The writing engine. This file:
- Uses Claude AI to write **all content in first-person AI voice** — "I am AlgoMind, I decided…".
- Generates **weekly portfolio review** posts (400–600 words) every Friday with trade walkthrough and SPY comparison.
- Generates **monthly deep dive** posts (600–900 words) on the 1st of each month — strategy analysis, ticker deep dives, forward look.
- Generates **milestone posts** (300–500 words) when portfolio hits key thresholds.
- Posts drafts via the Substack API (unofficial, session-auth via `SUBSTACK_TOKEN` cookie).
- **Falls back gracefully** to saving `.md` files in `drafts/` folder if Substack is not configured or API fails.
- **Status:** Complete. Substack posting optional — `SUBSTACK_TOKEN` + `SUBSTACK_PUB` env vars. Drafts always work locally.

#### `agent/agent_with_x.py`
The unified integration layer and primary entry point. This file:
- Wires `algomind_agent`, `x_poster`, and `substack_engine` together in a single `run_cycle()` function.
- `run_cycle()`: fetches data → Claude decides → Alpaca executes → Telegram notified → X posted → milestones checked → Substack milestone posts triggered.
- Separate scheduler with fine-grained 60-second polling loop — all events keyed to ET timezone (no reliance on system clock timezone).
- Handles morning outlook (9:30am), trade cycles (every 30 min), EOD events (4:05pm), Friday weekly recap, and 1st-of-month deep dive.
- In-memory state tracks what's already been posted today/this week to avoid duplicate posts after restarts within a session.
- **Status:** Complete. Run with `python agent/agent_with_x.py`.

### Architecture Notes
- All secrets via `.env` / environment variables — zero hardcoded credentials anywhere.
- Every external integration (Telegram, email, X, Substack) degrades gracefully if not configured.
- Claude is the **only** decision-maker — no hardcoded trading rules beyond position sizing guards.
- Starting capital: $50. Cash buffer: $2 minimum. Max single position: 30% of portfolio.
- Scheduler is fully timezone-aware (pytz / America/New_York) — safe to run on UTC servers.
